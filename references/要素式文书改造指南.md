# 要素式起诉状模板改造指南

> 基于交通事故案由实战经验总结，适用于任何案由的要素式文书改造

---

## 一、总体思路

**核心原则：基于原模板XML结构替换文本，绝不从零生成。**

要素式起诉状是法院制式文书，排版极其严格（字体、字号、行距、缩进、勾选框都有固定要求）。从零用python-docx生成几乎不可能100%还原样式。正确做法是：

1. 拿到法院提供的空白模板（或已填充实例的模板）
2. 解包docx → 读取document.xml → 定位每个待填充字段的XML位置
3. 用脚本精准替换文本，保留所有原有XML格式标签

---

## 二、改造步骤

### 第1步：模板XML结构分析

```python
import zipfile, xml.etree.ElementTree as ET

with zipfile.ZipFile('模板.docx') as z:
    with z.open('word/document.xml') as f:
        content = f.read().decode('utf-8')

ns = {'w': 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'}
tree = ET.fromstring(content)
body = tree.find('.//w:body', ns)
paragraphs = body.findall('.//w:p', ns)

for i, p in enumerate(paragraphs):
    texts = []
    for t in p.iter('{http://schemas.openxmlformats.org/wordprocessingml/2006/main}t'):
        if t.text:
            texts.append(t.text)
    full_text = ''.join(texts)
    if full_text.strip():  # 只看有内容的段落
        print(f'段落{i}: {full_text[:100]}')
```

**产出**：一张段落号→内容的映射表，标注每个待填充字段的位置。

### 第2步：字段定位与分类

把模板中所有待填充字段分三类处理：

| 类型 | 例子 | 替换策略 |
|------|------|----------|
| **简单文本替换** | 姓名、地址、金额 | 直接 `content.replace(old, new)` |
| **条件勾选框** | ☐/☑ | XML段落遍历+前序文本判断（见踩坑第1条） |
| **动态列表** | 证据清单、多个原告 | 拼接文本后替换占位内容 |

### 第3步：金额替换

金额在XML中是纯文本节点，直接字符串替换：

```python
content = content.replace('>393155.55<', f'>{total_amount:.2f}<')
```

**注意**：`>` 和 `<` 是XML标签的边界符，加上它们可以精确定位，避免误替换。

### 第4步：勾选框替换（最大坑点）

**❌ 错误做法**：用字符串搜索 `☐` 然后替换为 `☑`

**原因**：☐ 在XML中是独立的 `<w:t>☐</w:t>` 元素，它与前后文本之间隔了大量XML标签（间距可达243字符）。简单字符串搜索找不到它，或者搜索范围不够导致定位错误。

**✅ 正确做法**：XML段落遍历

```python
for p in root.findall('.//w:p', ns):
    texts = p.findall('.//w:t', ns)
    full_text = ''.join([t.text for t in texts if t.text])
    
    if '☐' not in full_text:
        continue
    
    for t in texts:
        if t.text != '☐':
            continue
        
        # 收集此☐之前的所有文本（前序文本）
        prefix = ''
        for pt in texts:
            if pt is t:
                break
            if pt.text:
                prefix += pt.text
        
        # 根据前序文本判断☐归属
        if '性别：男 ' in prefix:
            if gender == '男':
                t.text = '☑'
        elif '有 ' in prefix[-10:]:
            t.text = '☑'
        # ... 其他规则
```

**关键点**：
- ☐在模板中使用的是特殊字体（如MS Gothic），不要动字体设置
- 通过"前序文本"判断☐归属是最可靠的方式
- `prefix.endswith('有 ')` 注意rstrip会去掉尾部空格，导致匹配失败

### 第5步：打包输出

```python
with zipfile.ZipFile(output_path, 'w', zipfile.ZIP_DEFLATED) as zf:
    for root, dirs, files in os.walk(unpacked_dir):

```

## 三、踩坑清单

### 坑1：☐勾选框无法简单替换

- **现象**：用 `content.replace('☐', '☑')` 没有生效
- **原因**：☐是独立`<w:t>`元素，被大量XML标签包裹，字符串搜索范围不够
- **解决**：改为XML段落遍历，通过前序文本判断归属

### 坑2：rstrip()导致前序文本匹配失败

- **现象**：`prefix.rstrip().endswith('有 ')` 永远为False
- **原因**：rstrip去掉空格后字符串不可能以空格结尾
- **解决**：用 `prefix[-N:]` 截取末尾N个字符判断，不要rstrip

### 坑3：□（白色方框）与☐（空心方框）不同

- **现象**：模板中"女□"的□是普通文本字符，不是独立`<w:t>`元素
- **原因**：□（U+25A1）和☐（U+2610）是不同字符，在XML中的存在方式也不同
- **解决**：☐走XML遍历替换，□暂不处理（或作为后续优化项）

### 坑4：修改重复主事件时间必须同时改rrule

- 虽然这不是模板问题，但在Calendar系统中：修改重复日程的时间时，必须同时提供与新时间一致的rrule，否则系统会拒绝

### 坑5：日标准不同法院可能不同

- **现象**：有的法院住院伙食50元/天、营养20元/天，有的100/30
- **解决**：日标准设为可自定义参数，不要硬编码默认值

### 坑6：案件类型影响赔偿项目

- 刑事附带民事 → 不支持精神损害抚慰金
- 死亡案件 → 无残疾赔偿金，改为死亡赔偿金+丧葬费
- 仅财产损失 → 无医疗费、护理费等人身损害项目
- **解决**：用开关（case_type + injury_type）控制赔偿项目列表

### 坑7：xml.etree序列化丢失命名空间前缀

- **现象**：ET.tostring后命名空间前缀变成ns0/ns1，导致Word无法打开
- **解决**：用 `ET.register_namespace` 预注册命名空间

```python
NAMESPACES = {
    'w': 'http://schemas.openxmlformats.org/wordprocessingml/2006/main',
    'r': 'http://schemas.openxmlformats.org/officeDocument/2006/relationships',
}
for prefix, uri in NAMESPACES.items():
    ET.register_namespace(prefix, uri)
```
